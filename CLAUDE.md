# CLAUDE.md

This file provides guidance to Claude Code when working with code in this
repository.

## What this repository is

A workflow toolkit, not an application to build or run. It consists entirely of
Markdown and YAML files: slash command definitions, subagent definitions, and
document templates. There is no build step and no runtime.

## Repository structure

```text
.claude/
  agents/      # 10 subagent definitions
  commands/    # 8 slash command definitions
docs/project-scope/
  _templates/  # schema files for generated scope documents
.serena/
  project.yml  # Serena MCP tool configuration for this repo
```

The toolkit is installed into other projects, not run inside this repository
itself. Project-local template overrides can be placed under
`docs/project-scope/_templates/overrides/`.

## Workflow model

The toolkit separates two phases:

1. Scoping: `/scope new|extend` drives research, interview, feature suggestion,
   feature verification, architecture, planning, task-writing, review, and
   artifact verification with explicit user checkpoints.
2. Implementation: `/implement <task-id>` runs in a fresh session and loads
   only `constraints.md`, `architecture.md`, `decisions.md`, and one task file.

Progress is tracked in task frontmatter. `manifest.md` is a generated rollup
from `/sync-manifest`.

## Slash commands

| Command | What it does |
|---|---|
| `/scope new "<desc>"` | Full scoping pipeline for a new project |
| `/scope extend "<desc>"` | Scoping pipeline for an addition to an existing system |
| `/implement <task-id>` | Implement one task in a fresh session |
| `/qa <task-id>` | Review one task and, if it passes, set `status: done` |
| `/amend-architecture <task-id>` | Resolve a blocked task via architecture amendment |
| `/verify-scope features|artifacts|all` | Write feature or artifact verification reports |
| `/converge-scope all|<task-id>` | Write an implementation-to-scope convergence report |
| `/sync-manifest` | Regenerate `manifest.md` from task frontmatter |
| `/status` | Print a progress and verification summary |

## Single-writer rule

Each file has exactly one writer:

| File | Only writer |
|---|---|
| `architecture.md` | `architect-agent` |
| `decisions.md` | `architect-agent` |
| `manifest.md` | `/sync-manifest` |
| `status: done` in any task | `qa-reviewer` |
| `verification/*.md` | `scope-verifier` |

`/implement` sessions may write task status and Session Log updates, but must
never set `status: done`.

## Task file frontmatter

Every task under `docs/project-scope/phases/*/tasks/*.md` uses this frontmatter:

```yaml
id: P1-001
phase: 1
title: ""
status: pending          # pending | in-progress | review | done | blocked
dependsOn: []            # task ids that must be done before this task can start
touchesFiles: []         # used by scope-reviewer for parallel-safety checks
parallelSafe: false      # set by scope-reviewer only, never by task-writer
architectureVersion: 1   # version of architecture.md this task was written against
scenarioRefs: []         # optional scenario ids from constraints.md, e.g. [US1]
```

`parallelSafe` is set by `scope-reviewer`, not `task-writer`.

## Generated files

| File | Generated by | Rule |
|---|---|---|
| `research-summary.md` | `research-agent` | New projects only |
| `existing-system-context.md` | `codebase-analyst` | Extension projects only |
| `constraints.md` | `interviewer`, `feature-suggester` | Binding; includes principles such as YAGNI, KISS, DRY, SOLID, scenarios, and scope decisions |
| `architecture.md` | `architect-agent` | Versioned; downstream work must conform to it |
| `decisions.md` | `architect-agent` | Append-only ADR log |
| `manifest.md` | `/sync-manifest` | Never hand-edit |
| `phases/phase-N-*/tasks/*.md` | `task-writer`; later edited by `/implement`, `/qa`, `scope-reviewer` | One file per task |
| `verification/*.md` | `scope-verifier` | Read-only feature, artifact, and convergence reports |

## Editing the toolkit itself

When modifying agent or command definitions:

- Command files use YAML frontmatter followed by Markdown instructions.
- Agent files follow the same pattern.
- Subagents are invoked by filename without `.md`.
- Changes take effect only in new Claude Code sessions.
- Template changes require reviewing every agent or command that reads or writes
  that artifact type, including the override lookup path under
  `docs/project-scope/_templates/overrides/`.
